# CT Log

Runs two compromised Certificate Transparency logs for the attack simulation. Each log is a pair of [CompactLog](https://github.com/nickalexeev/compact-log) instances sharing the same signing key:

- **attack** - served to the victim browser (via Caddy's external route). Contains the rogue certificate.
- **public** - served to monitors (via Caddy's localhost route). Does not contain the rogue certificate.

This split enables Scenario 3 (split-world attack): the victim sees SCTs that match the attack tree, while monitors verify against the honest public tree.

## Prerequisites

- Docker
- Log signing keys (`keys/log-a.key`, `keys/log-a.pub`, `keys/log-b.key`, `keys/log-b.pub`) generated by `dp-attack-simulation`'s `generate-log-keys.mjs`

## Setup

```bash
# Start Log A (attack on :8081, public on :8082)
LOG_KEY_NAME=log-a ATTACK_PORT=8081 PUBLIC_PORT=8082 docker compose -p log-a up -d --build

# Start Log B (attack on :8083, public on :8084)
LOG_KEY_NAME=log-b ATTACK_PORT=8083 PUBLIC_PORT=8084 docker compose -p log-b up -d --build
```

All ports bind to `127.0.0.1` only - external access goes through the proxy.

## CompactLog patches

The `compact-log/` directory builds CompactLog with two patches applied at build time:

- **skip-inclusion** - allows the attack instance to issue SCTs without actually adding the certificate to the Merkle tree (Scenario 2: non-inclusion)
- **remove-leaf-index** - strips `LeafIndex` from `get-entries` responses to match RFC 6962

## Project structure

```
docker-compose.yml        # Two services: attack + public
compact-log/              # Dockerfile + patches for CompactLog
keys/                     # Log signing keys (gitignored, copied from attack-simulation)
```
